# 产品需求文档 (PRD): “AI+制造业” 独立站作品集

## 1. 产品概述 (Product Overview)
**产品名称**：Beam Liu 个人数字作品集 (Beam Portfolio)
**产品定位**：一个打破传统 PDF 简历格式，高度互动、具有科技感与商业高级感的个人独立站点。
**核心目标**：
1. 全方位展示 Beam Liu 作为“底层技术极客 + 高级商业售前顾问 + 敏捷研发管理者”的多面手（六边形战士）形象。
2. 吸引并打动目标公司的面试官、高管及业务线负责人（重点针对“AI+制造业”售前岗或研发管理岗）。
3. 整合散落的个人多媒体资产（B站视频、Baidu Netdisk 售前 PPT、官网链接、PDF/图片等），一站式呈现。

## 2. 目标用户与痛点 (Target Audience & Pain Points)
### 2.1 目标用户画像
*   **企业高管 / 业务负责人**：关注商业价值、行业痛点解决能力、售前转化率。
*   **技术 VP / 架构师**：关注底层技术深度（K8s, PyTorch, LLM）、微服务架构经验、技术创新（专利）。
*   **HR 招聘人员**：关注经历匹配度、团队管理规模、背景与认证。

### 2.2 用户痛点 (PDF简历的局限性)
*   **无法展示动态能力**：PDF 无法播放西门子售前认证视频、无法展示高频互动的 EMOP Demo。
*   **排版受限**：19 年的丰富经历（Dell, OOCL, 易行 CTO）与 12 项专利挤在纸面上缺乏吸引力。
*   **认知维度单一**：难以立体展示作为“技术极客”的阅读量（250本书）、天文哲学爱好，容易被标记为纯“敲代码的”。

## 3. 核心功能与模块需求 (Core Features)

### 3.1 首屏概述 (Executive Summary Section)
*   **需求描述**：提供沉稳、专业的个人品牌第一印象。
*   **功能需求**：
    *   **视觉风格**：极简高级浅色主题 (Light Theme)，采用大面积留白与干净纯白背景。
    *   **排版约束 (新)**：以宽幅单列 (Single-Column) 为主，确保在 1:1 实际比例下文字硕大清晰，避免分列造成的细碎感。
    *   **头部视觉元素 (Header Elements)**：
        1. **专业头像 (Avatar)**：在首屏/Header区域醒目位置融入高质量个人职场头像。
        2. **隐形水印 (Subtle Watermark)**：在 Header 背景中，添加极低透明度的浅灰色科技 / 几何线条水印底纹。
    *   **核心 Slogan**：“链接前沿 AI 与数字化制造的工程师 / 管理者 / 破局者”。
    *   **核心数据指标**：以清晰、端庄的排版展示核心数据。

### 3.2 核心价值矩阵 (Value Proposition Matrix)
*   **需求描述**：将能力沉淀为针对“售前”和“管理”岗位最具杀伤力的优势。
*   **功能需求**：采用响应式卡片网格布局，展示：
    1.  **制造业全栈理解**：从 PLM 到 MOM/MES，深谙供应链与生产线痛点。
    2.  **AI/大模型原力**：Google DL/ML 双重认证，从模型训练到工业场景应用。
    3.  **售前与商业转化**：不仅懂技术，更懂客户。1小时拿下700万订单的破局能力，出色的 PPT 与宣讲能力。
    4.  **敏捷梯队建设**：带领 70+ 团队，推崇氛围编程，敏捷开发布道者。

### 3.3 深度商业案例与履历剖析 (Career Trajectory & Case Studies)
*   **需求描述**：对于总监/高管及资深售前而言，简单的列表和缩略图画廊显得过于单薄。需要用“商业白皮书”的质感，将“履历”与“主导的千万级项目”深度融合展示。
*   **功能需求**：
    *   **高管级职业轨迹 (Executive Trajectory)**：摒弃由简单圆点组成的传统简历时间轴。采用宽幅的网格线布局（Bento Grid 或对齐的文本块）。清晰标出在每一任期（如易行 CTO，Dell Dev Manager）的核心管理规模、技术路线图演进。
    *   **深度商业案例展厅 (In-Depth Case Studies)**：这是取代简单画廊的重头戏。每一个大项目（商飞 MOM、大族激光 PLM、EMOP平台）作为一个独立的**案例卡片 (Case Study Card)**。
        *   **排版模式**：左图右文（或上图下文）。
        *   **内容结构**：必须包含具体的 **“业务挑战 (Challenges)”** 和 **“商业收益 (Impact/ROI) - 如降低成本20%”**。这正是 JSON 简历中最具含金量的部分。
        *   **多媒体集成**：在每个案例卡片的操作区，提供极简的 `[ ▷ 播放架构讲解 ]` 或 `[ 📄 查看售前方案 PDF ]` 按钮，点击后平滑展开。      *   提供文档（如 PPT/PDF）的清晰下载入口或在线预览。

### 3.4 专业认证与硬核专利 (Certifications & Patents)
*   **需求描述**：以权威、专业的方式展示学术与知识产权成果。
*   **功能需求**：
    *   **专利列表**：摒弃花哨的弹幕，采用类似高质量学术论文网站的排版结构，清晰列出专利号和名称。
    *   **认证标识**：Google 与 Siemens 认证章以灰度或标准微章形式精致陈列。

### 3.5 认知与视野 (Beyond Code)
*   **需求描述**：用个人生活维度拉满好感度与“软实力”。
*   **功能需求**：
    *   结构化呈现书籍阅读版图（技术、哲学、经管、科幻），采用现代主义的网格或极简柱状图。
    *   专业且克制地展示个人爱好及足迹。

## 4. 非功能性需求 (Non-Functional Requirements)
*   **端侧适配 (Responsive Design)**：完美适配 PC 与移动端。
*   **性能 (Performance)**：极致的加载速度，媒体资源懒加载。
*   **视觉美学 (Aesthetics - Minimalist & Professional)**：
    *   **摒弃光污染**：完全放弃霓虹色、复杂粒子动画和炫光。
    *   **ToB 商业级高级感**：参考 Apple, Vercel, 或 Stripe 的企业介绍页。结构清晰、大巧不工、字体优雅、留白得当。依靠排版和内容本身的力量打动高管。
*   **SEO 与分享**：设置完善的 Meta 标签，确保在微信或 LinkedIn 抛出链接时，有精美的预览图（OG Image）和标题。
